# Data Model

> Authoritative entity reference for the Platform. Field types match the Postgres migrations under [`supabase/migrations/`](../../supabase/migrations/). Every table is **RLS-enabled** with the policy outline in [`AUTH-AND-RLS.md`](AUTH-AND-RLS.md).

## 1. Conventions

- Primary keys: `uuid` generated by `gen_random_uuid()`, column `id`.
- Timestamps: `created_at timestamptz not null default now()`, `updated_at timestamptz not null default now()` (kept in sync by a trigger).
- Soft delete: `deleted_at timestamptz null`. No row is ever hard-deleted from the catalog or sales tables.
- Money: integer cents. Column type `bigint`. Column suffix `_cents`. Currency lives in a sibling `currency` column (`USD` at launch).
- Identifiers from external systems (Stripe, suppliers) live in dedicated columns prefixed with the provider (`stripe_session_id`, `supplier_external_id`).
- Enum-like columns use Postgres `enum` types declared in the first migration that needs them.

## 3. Pricing & tiers

### `tiers`
| Column | Type | Notes |
|---|---|---|
| `id` | `uuid` ||
| `code` | `text` | `'tier_1'`, `'tier_2'`, `'tier_3'`, `'tier_4'`; unique |

### `price_rules`
A composable rule. The pricing engine evaluates the matching set per cart line.

| Column | Type | Notes |
|---|---|---|

| `priority` | `int` | higher wins on conflict |

### `prices`
Materialized effective prices per variant × tier, refreshed when `price_rules` change.

| Column | Type | Notes |
|---|---|---|
| `variant_id` | `uuid` ||
| `tier_id` | `uuid` ||
| `price_cents` | `bigint` ||
| `currency` | `text` | `'USD'` |
| primary key | `(variant_id, tier_id)` ||

## 9. Indexes & invariants

- `unique (slug)` on `products`, `unique (sku)` on `product_variants`.
- `unique (stripe_payment_intent_id)` on `payments`.
- `check (qty >= 1)` on `cart_items` and `order_items`.
- `check (price_cents >= 0)` on `prices`, `cart_items`, `order_items`.
- `check (available_qty >= 0)` on `inventory_snapshots`.
- Partial index `idx_orders_active on orders (account_id) where status not in ('canceled','refunded')`.
- BRIN index on `audit_log.created_at` for time-window queries.
